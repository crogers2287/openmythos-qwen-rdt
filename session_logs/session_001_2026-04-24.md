## Unsloth Attempts (v1-v11) — ALL FAILED
| Version | Approach | Result |
|---------|----------|--------|
| v1-v7 | RecurrentDepthWrapper calling layers individually | Hung (per-layer gradient checkpointing explosion) |
| v8 | Monkey-patch text model forward | TypeError: wrong output format |
| v9 | Fix attention mask (set None) | TypeError: output not mapping |
| v10 | Return BaseModelOutputWithPast | AttributeError on spectral radius |
| v11 | Fix PEFT-wrapped model navigation | Hung (gradient checkpointing deadlock) |
| SFT | Clean Unsloth SFTTrainer, no RDT loop | **WORKED** but ~50h ETA, no RDT |

### Root Cause
Unsloth patches `__call__` on every decoder layer with gradient checkpointing, compiled Triton kernels, and accelerate hooks. Calling layers in a custom loop order deadlocks or hangs. This is fundamental incompatibility, not a bug we can work around.

### Decision
**RDT requires non-Unsloth training** (Option B). Use plain Transformers + PEFT.

## Hard Reset — Profile-First Approach (Issue #1)
User directive: stop guessing, measure reality, work backward from 24h constraint.

### Corrected Compute Model
```
RDT effective depth = Prelude(4) + Core(4) × R + Coda(4) = 8 + 4R
This is EXECUTION count, not unique layer count.

R=2  → 16 executions → 0.40x baseline
R=4  → 24 executions → 0.60x baseline
R=8  → 40 executions → 1.00x baseline
R=16 → 72 executions → 1.80x baseline
```

### Architecture Finding
Core layers 16-19 = 3 DeltaNet (cheap, no KV cache) + 1 full attention (O(T²) per loop, recomputed each iteration). No KV cache reuse across recurrence.

### Backward Pass Cliff
R=2 → R=4 backward time jumps from 3.4s to 25.5s (7.5x). VRAM goes from 73GB to 77GB. At 77GB, activation storage pressure causes massive backward pass slowdown.

## Key Lessons
1. **You cannot modify layer execution order under Unsloth.** Its gradient checkpointing, Triton kernels, and accelerate hooks are deeply integrated. Custom forward paths deadlock.
2. **Measure before building.** Forward-only profiling ≠ training profiling. The backward pass cost cliff between R=2 and R=4 was invisible without profiling.
3. **Execution count, not layer count.** RDT at R=16 is 1.8x MORE compute than baseline, not less. The recurrence multiplier dominates.
4. **Ephemeral storage on Packet.ai wipes on container restart.** Use persistent `/data/share311/` for code/venv, ephemeral for HF cache.
5. **Qwen3.6 loads as different model types depending on loader.** Unsloth: `Qwen3_5MoeForConditionalGeneration` (multimodal). Plain transformers: `Qwen3_5MoeForCausalLM` (text-only). Different internal paths.
